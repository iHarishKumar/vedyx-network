⚙️ How It Works

1️⃣ Detection Phase (Reactive Network)

Reactive Network captures events from origin chains → VedyxExploitDetectorRSC routes to detectors → Pattern analysis (large transfers, mixer usage) → Callback to destination chain

2️⃣ Validation Phase (Community Voting)

Suspicious address received → Check verdict history → Repeat offender? Auto-mark (87% gas savings) → New/Clean? Community voting → Stake-weighted votes → 7-day voting period → Finalization & verdict recording

3️⃣ Risk Assessment Phase

Calculate risk score: Verdict (0-40) + Incident history (0-20) + Detector severity (0-20) + Voting consensus (0-10) + Time recency (0-10) = Total Score (0-100)

4️⃣ Integration Phase (DeFi Protocols)

Uniswap V4 calls hook → Query risk engine → Apply dynamic fees (1-30%) or block HIGH/CRITICAL users → Transaction executes with adjusted parameters

🛡️ Anti-Manipulation Design

Can't attackers just buy votes to mark themselves as "not suspicious"?

Economic Disincentives Make Vote Buying Unfeasible

Stake-Weighted Voting: More stake = More voting power
To Manipulate Votes: Attackers need MASSIVE stake
If They Acquire Stake: They've bought into the system & become economically aligned
Incorrect Voters: Lose 10% of stake
Negative Karma: Exponentially reduces voting power
Below -50 Karma: Blocked from voting entirely

💡 The Cost-Benefit Analysis

Cost to Manipulate: Acquire majority stake + Risk 10% penalty + Karma damage
Benefit: Mark one address as "not suspicious"
Result: It's cheaper to just not exploit in the first place!
